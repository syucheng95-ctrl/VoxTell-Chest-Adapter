# 胸部专用融合模块：最小实现方案

本文档从 [改进方案.pdf](/C:/Users/Sunyucheng/Desktop/作业(大学相关）/生医工大赛/题目2-demo/改进方案.pdf) 中挑选当前最适合本项目落地的一条路线，并压缩成最小可实现版本。

## 1. 选型结论

不建议当前直接做：

- `Token-to-Voxel Cross-Attention`
- `3D Deformable Cross-Attention`
- `Tree-Guided Cascaded Fusion`

原因：

- 改动面太大
- 需要更完整训练链路
- 对显存与调参要求更高
- 容易让项目从“改进 VoxTell”变成“重做新模型”

当前建议优先做的方向是：

- `Text-Guided Adapter / 胸部专用融合模块`

这是因为它：

- 与现有 VoxTell 结构最兼容
- 改动最轻
- 更适合当前资源条件
- 更容易在比赛答辩里讲清楚“为什么针对胸部 CT 有意义”

## 2. 当前要解决的核心问题

从现有实验看，当前系统的问题不是单纯“不会响应文本”，而是：

- `raw prompt` 稳，但不够专门化
- `structured prompt` 会改变行为，但不稳定
- `LoRA` 会提升 recall，却容易大面积过分割

所以当前最该优化的点是：

- 提高 text-to-vision 融合的针对性
- 抑制无约束扩张
- 优先保 precision，而不是盲目提 recall

## 3. 最小实现目标

### 目标一句话

在不改 VoxTell 主体框架的前提下，在视觉编码侧插入一个胸部专用的文本引导适配模块，让文本信息对 3D 特征产生受控调制。

### 设计原则

- 不动 `Qwen` 本体
- 不重写 `encoder`
- 不重写 `decoder`
- 不引入重型 cross-attention
- 只在中间特征上做轻量调制

## 4. 最小模块设计

建议做一个 `ChestTextGuidedAdapter`，插在 encoder 输出到 decoder 输入的桥接位置，或者插在 `project_to_decoder_channels` 前。

### 输入

- 3D 视觉特征 `F`
- 文本 embedding `T`

### 输出

- 调制后的视觉特征 `F'`

### 最小形式

先做最简单的 FiLM / gated adapter 版本：

```text
gamma, beta = MLP(T)
gate = sigmoid(Linear(T))
F' = F + gate * (gamma * F + beta)
```

解释：

- `gamma` 控制通道缩放
- `beta` 控制偏移
- `gate` 控制本次文本是否强力干预视觉特征

这个版本的优点是：

- 比 LoRA 更明确地控制文本如何影响视觉特征
- 比 cross-attention 轻得多
- 非常适合先做最小原型

## 5. 推荐插入位置

当前优先级建议如下：

1. `project_to_decoder_channels` 之前
2. `transformer_decoder` 输入前
3. 多尺度版本作为后续扩展

当前最小版本只做一个插入点，不做多处堆叠。

原因：

- 改动越少，越容易判断是否有效
- 一开始不需要做复杂多尺度融合
- 先看单点 adapter 能否比 LoRA 更稳

## 6. 训练策略建议

### 冻结范围

冻结：

- `Qwen`
- `encoder`
- 原卷积 `decoder`

可训练：

- 新增的 `ChestTextGuidedAdapter`
- 必要时保留 `project_text_embed`

### 为什么这样做

- 这样训练参数量远小于大规模改主干
- 可以把收益更明确地归因到“融合模块”
- 更符合“比赛可讲、工程可落地”的要求

## 7. 第一阶段验证目标

第一阶段不要直接上 full val。

先做两个层次：

### 阶段 A：最小可运行

- 能成功插入模块
- 推理不崩
- loss 能下降

### 阶段 B：小规模验证

- 用 5-case 或 10-case 验证
- 只比较：
  - `raw baseline`
  - `raw + Chest Adapter`

### 阶段 C：再做 full val

只有当小规模验证出现清晰正向信号时，再上 full val。

## 8. 成功判据

当前这条改进线最重要的不是把 recall 顶高，而是：

- `precision` 不明显塌陷
- `Dice` 至少不低于 `raw baseline`
- 输出不出现 LoRA 那种大面积无约束膨胀

可以接受：

- recall 小幅上升或持平

不可以接受：

- precision 大幅下降
- 大量病灶外误分割

## 9. 代码落地建议

建议新增如下文件，而不是直接大改原有核心文件：

- `scripts/training/train_voxtell_adapter.py`
- `scripts/analysis/evaluate_voxtell_adapter.py`
- `VoxTell/voxtell/model/chest_text_guided_adapter.py`

当前不建议做的事：

- 不要一开始就把 adapter 散布到所有尺度
- 不要同时加 prompt routing
- 不要同时再叠 LoRA

## 10. 为什么这条线比继续 LoRA 更合适

### LoRA 当前的问题

- 学到的是高召回低精度
- 输出校准差
- 容易过分割

### Adapter 路线的优势

- 文本如何作用于视觉特征是显式的
- 更容易加入 gate，限制过强调制
- 更容易在答辩里讲“胸部专用融合”

换句话说：

- LoRA 更像“参数层面的软调整”
- Adapter 更像“结构层面的可解释增强”

## 11. 当前执行版

当前最值得开始的版本只有一个：

### 版本名

`ChestTextGuidedAdapter v1`

### 范围

- 单插入点
- 单层 MLP 生成 `gamma / beta / gate`
- 只对 `raw prompt` 路线验证

### 第一轮对比

- `raw baseline`
- `raw + ChestTextGuidedAdapter v1`

### 暂缓

- `structured + adapter`
- `hybrid + adapter`
- token-level cross-attention
- deformable attention
- anatomical tree fusion

## 12. 一句话决策

当前最适合本项目的“改进方案落地版”不是重做 VoxTell，而是：

先做一个轻量、可解释、胸部专用的 `Text-Guided Adapter`，用最小改动验证它能否比当前 LoRA 更稳。

## 13. 当前实现状态与结论

`ChestTextGuidedAdapter v1` 已经完成了第一轮完整实现、训练和 full val。

已完成内容：

- 新增 `VoxTell/voxtell/model/chest_text_guided_adapter.py`
- 在 `VoxTellModel` 中插入单点 `Text-Guided Adapter`
- 完成训练脚本 `scripts/training/train_voxtell_adapter.py`
- 完成推理脚本 `scripts/experiments/run_voxtell_val_adapter_raw.py`
- 完成评估脚本 `scripts/analysis/evaluate_voxtell_val_adapter_raw.py`

full val 结果：

- `raw baseline`
  - `mean_dice = 0.2139`
  - `micro_dice = 0.4228`
  - `mean_precision = 0.2493`
  - `mean_recall = 0.3099`
- `raw + ChestTextGuidedAdapter v1`
  - `mean_dice = 0.0912`
  - `micro_dice = 0.2422`
  - `mean_precision = 0.0657`
  - `mean_recall = 0.4380`

解释：

- `v1` 依然是高召回、低精度、过分割严重。
- 这说明“单点 gated FiLM adapter + 原损失函数”不足以解决当前问题。
- 当前不应继续只靠加训练时长来补救 `v1`。

因此，这份最小实现方案的角色已经从“准备实现”变成了“已验证失败的第一版原型”。下一步若继续沿用 `Text-Guided Adapter` 主线，至少需要增加一项更强约束：

- 更强的假阳性抑制损失
- 更严格的负样本采样
- 更保守的 gating 形式
- 或显式的解剖/区域先验约束

## 14. v2 修正与结果

在 `v1` 失败后，已经实现并验证了 `ChestTextGuidedAdapter v2`。

`v2` 的核心修正：

- 插入点从 `post_decoder` 前移到 `pre_decoder`
- 调制从全通道大幅残差改为分组 gate + 小幅残差
- 训练时提高负样本 BCE 权重
- 对负样本 patch 增加更强的假阳性惩罚

结果文件：

- [outputs/voxtell_val_adapter_raw_v2_metrics.json](/C:/Users/Sunyucheng/Desktop/作业(大学相关）/生医工大赛/题目2-demo/outputs/voxtell_val_adapter_raw_v2_metrics.json)

聚合结果：

- `raw baseline`
  - `mean_dice = 0.2139`
  - `micro_dice = 0.4228`
- `raw + ChestTextGuidedAdapter v2`
  - `mean_dice = 0.2206`
  - `micro_dice = 0.4099`

解释：

- `v2` 已经把 `mean_dice` 提高到高于 baseline，说明修正方向有效。
- 但 `micro_dice` 仍略低，说明总体体素级误报仍需继续压缩。
- 因此，当前最合理的主线不是推倒重来，而是在 `v2` 基础上继续做“precision / micro”导向优化。
