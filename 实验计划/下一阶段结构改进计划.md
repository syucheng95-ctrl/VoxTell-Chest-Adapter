# 下一阶段结构改进计划

本文档用于明确 `adapter v2` 之后的下一阶段改进方向，重点回答三件事：

1. 现在结果处于什么状态
2. 后面结构上还能怎么改
3. 最值得先做的版本是什么

## 1. 当前结果状态

当前最佳版本是 `ChestTextGuidedAdapter v2`。

核心结果：

- `raw baseline`
  - `mean_dice = 0.2139`
  - `micro_dice = 0.4228`
  - `mean_precision = 0.2493`
  - `mean_recall = 0.3099`
- `adapter v2`
  - `mean_dice = 0.2206`
  - `micro_dice = 0.4099`
  - `mean_precision = 0.2435`
  - `mean_recall = 0.3461`

结论：

- `mean_dice` 已经打过 baseline
- `recall` 也更高
- 但 `micro_dice` 还略低
- 因此当前问题已经从“方向对不对”变成“怎么把总体误报再压下去”

## 2. 为什么 v2 会比 v1 好

`v2` 的提升不是偶然，而是因为它做了三类正确修正：

### 2.1 更早介入

`v1` 是在 `post_decoder` 调制，也就是在 `transformer_decoder` 输出后再改 `mask_embedding`。

`v2` 改成了 `pre-decoder`：

- 先做 `project_text_embed`
- 再进入 `text_guided_adapter`
- 然后进入 `transformer_decoder`

这意味着文本是在“形成响应之前”参与，而不是“快出结果时整体放大”。

### 2.2 更保守的调制方式

`v2` 不再做大幅的全通道调制，而是：

- 分组 gate
- 小残差
- gate 上限限制
- `tanh` 限幅

这让文本更像“微调方向”，而不是“猛踩油门”。

### 2.3 更强的负样本约束

`v2` 在训练中加入了：

- 更高的负样本 BCE 权重
- 更强的 negative patch loss scale
- 额外的假阳性惩罚项

这正好对应了前面失败版本暴露出来的问题：假阳性压不住。

## 3. 后面模型结构还能怎么改

后续结构改动建议分三档。

### 3.1 第一档：低风险、最值得先做

这些改法最适合当前项目阶段。

#### A. 类别感知 adapter

问题：

- 不同病灶类型的误报模式不同
- 现在一套 gate 管所有类别，过于粗糙

改法：

- 为不同类别设置不同的小头
- 或在 adapter 中加入类别 embedding
- 让 `nodule / GGO / consolidation / effusion` 走不同的调制偏置

预期收益：

- 更贴合医学任务
- 更有机会减少“统一策略带来的误报”

#### B. suppression branch

问题：

- 现在的 adapter 仍然主要是“怎么增强”
- 缺少一个明确的“哪里应该压掉”的结构

改法：

- 新增一个背景抑制分支
- 输出一个 suppression signal 或 suppression gate
- 与主 adapter 输出结合，专门抑制病灶外区域

预期收益：

- 直接针对 `micro_dice` 偏低的问题
- 有助于把误报再往下压

#### C. 多尺度 adapter

问题：

- 现在只在一个点调制
- 小目标和大目标依赖的尺度不一样

改法：

- 保留当前 `pre-decoder` 主 adapter
- 再在更高分辨率尺度增加一个更弱的小 adapter
- 只做 1 个附加尺度，不要一次做太多

预期收益：

- 对小结节类目标更友好
- 比单点 adapter 更细

### 3.2 第二档：中等风险，但可能更强

#### D. token-level text fusion

现在主要还是整句 embedding。  
可以继续往前走，把 token 级信息保留一部分，让部位词、病灶词、形态词不要完全混成一个向量。

适合什么时候做：

- 当 `v3` 已经把 `micro_dice` 拉近或拉平 baseline
- 需要更强论文点时

#### E. region-aware fusion

根据文本里的粗部位信息，先形成粗区域约束，再让文本只在相关解剖区域里调制特征。

例如：

- `right lower lobe`
- `left upper lobe`
- `subpleural`

这类约束非常适合胸部 CT，且比重型 cross-attention 更容易落地。

### 3.3 第三档：高风险，不建议当前优先做

#### F. token-to-voxel cross-attention
#### G. 3D deformable cross-attention
#### H. anatomy/tree-guided cascaded fusion

这些方向不是没价值，而是当前阶段代价太高：

- 工程复杂
- 显存压力大
- 调参周期长
- 容易把项目带成“新模型开发”

当前不应优先做。

## 4. 最值得先做的下一版

当前我建议的下一版是：

## `ChestTextGuidedAdapter v3`

结构定义：

- 保留 `pre-decoder`
- 保留保守 residual gate
- 加入 `category-aware gate`
- 加入 `suppression branch`

目标：

- 保住 `mean_dice > baseline`
- 把 `micro_dice` 推过 baseline
- 不再走“只靠提 recall”这条路

## 5. 推荐执行顺序

### 第一阶段

先做：

- `category-aware gate`
- `suppression branch`

先不要做：

- token-level cross-attention
- 多尺度大改
- 解剖树结构

原因：

- 改动面可控
- 更容易归因
- 训练和验证链路可以直接复用

### 第二阶段

如果 `v3` 有正向信号，再做：

- 轻量多尺度 adapter
- 粗区域约束

### 第三阶段

只有在前两步都站稳后，再考虑：

- token-level text fusion

## 6. 实际落地建议

建议下一步新增或修改的代码方向：

- `VoxTell/voxtell/model/chest_text_guided_adapter.py`
  - 扩展成 `v3` 结构
- `VoxTell/voxtell/model/voxtell_model.py`
  - 保留 `pre-decoder` 接线
  - 接入 `suppression branch`
- `scripts/training/train_voxtell_adapter.py`
  - 加入类别感知配置
  - 记录 suppression 相关 loss

## 7. 一句话结论

当前最值得继续的不是回头补 `LoRA`，也不是直接冲重型 cross-attention，而是在 `adapter v2` 的基础上做 `v3`：

让文本不只会“加强哪里像病灶”，还会“抑制哪里不该分”。  
这才是把 `micro_dice` 推过 baseline 的最合理路径。
